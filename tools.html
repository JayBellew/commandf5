Accessibility Tools

Google Accessibility

Guides and resources for implementing and managing accessibility tools and practices in your company. Information and resources to help you develop accessible products and apps.

Hike By Facebook

The primer teaches some essentials of web accessibility through simple exercises. HIKE is an acronym to help people remember the basic principles of writing accessible code for the web

H: stands for headings and semantic markup.
I: stands for images and labels.
K: stands for keyboard navigation.
E: asks for you to ACT with a little extra love for custom components and more.

WAVE by WebAIM

WAVE is a suite of tools for facilitating web accessibility evaluation by providing a visual representation of accessibility issues within the page. There is an online service, Firefox and Chrome extensions, and an online and installable API engine.

Accessibility Bookmarklets by University of Illinois and Pixo

Provides visualizations of accessibility features of web pages like headings, landmarks, images, links and form controls. Provides computed information on the role, accessible name and accessible descriptions for accessible elements that are visualized.

Chrome Browser Accessibility Tools by Google Accessibility

Adds an Accessibility audit, and an Accessibility sidebar pane in the Elements tab, to your Chrome Developer Tools.

Accessibility Viewer by The Paciello Group

The Accessibility Viewer (aViewer) is an inspection tool for Windows that displays the accessibility API information (MSAA, IAccessible2, UI Automation, ARIA, HTML DOM) exposed by web browsers to the operating system, and thus to any assistive technology (AT) such as screenreaders.

Accessibility for Developers by Apple

With accessibility APIs in iOS, macOS, and watchOS, you can make your apps seamlessly functional for a broad range of users.

Accessible Colour Evaluator by daprlab

With a user-centered design approach, we developed the Accessible Colour Evaluator (ACE) which enhances web developers’ and designers’ ability to balance aesthetic and accessibility constraints for choosing a website’s colour scheme.

AccessLint CI by Thoughtbot, Inc

AccessLint CI automatically comments on your pull-requests instead of triggering build failures, and only tells you what has changed since the last build. AccessLint CI makes it easy to catch the obvious issues, and is the perfect companion to manual accessibility review of more complex issues that

Visual ARIA by Bryan Garaventa

Visual ARIA allows engineers, testers, educators, and students to physically observe ARIA usage within web technologies, including ARIA 1.1 structural, live region, and widget roles, proper nesting and focus management, plus requisite and optional supporting attributes to aid in development.
